# TASK: 핫픽스 2건

## T1. 더보기 페이지네이션 — 1개만 나오고 중복 처리

더보기 누르면 1개만 추가되고 "중복 광고가 많습니다. 더보기를 다시 눌러주세요" 토스트 표시.
55건 중 30건 표시 후 더보기 누르면 나머지 25건이 한번에 나와야 한다.

SearchAPI.io는 한 페이지당 30건씩 반환한다. 중복 제거 로직이 너무 공격적으로 필터링해서 거의 다 버리는 것으로 보인다.

### 확인할 것
1. SearchAPI.io 응답에서 실제로 몇 건이 오는지
2. 중복 제거 기준이 뭔지 (ad_archive_id? 전체 비교?)
3. 이미 표시된 30건과 새로 온 30건 사이 중복이 실제로 몇 건인지
4. 중복 제거 후 남는 건수가 왜 1건뿐인지

### 수정 방향
- 중복 제거는 ad_archive_id 기준으로만
- 중복이 있어도 새 광고가 있으면 추가 표시
- "중복 광고가 많습니다" 토스트 제거 — 사용자한테 보여줄 메시지가 아님

## T2. 모니터링 패널 — 인스타 아이디 표시

모니터링에 등록된 브랜드 아래에 @인스타아이디가 나와야 한다.
현재 "젝시믹스 @xexymix"는 나오는데, "올리브영"은 @아이디 없이 이름만 나옴.

### 수정
- 모니터링 등록 시 ig_username도 함께 저장
- 모니터링 패널에서 브랜드명 아래에 @ig_username 표시
- ig_username이 null이면 표시 안 함 (이름만)

## 검증
- 더보기 클릭 → 25건 추가 로드 (총 55건)
- 모니터링 패널에 @인스타아이디 표시
- npm run build 성공
- 커밋+푸시까지
