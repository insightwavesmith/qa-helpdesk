## T6: 게이지 목업 디자인 일치

### 현재 동작
- 등급 영역: 파스텔 톤 (흐림)
- 포인터: 중심에서 뻗는 시계 바늘 (line + pivot circle)
- 파란색 진행 아크 있음 (목업에 없는 요소)
- 0/50/100 수치 마커 있음 (목업에 없음)
- 배경 회색 호 없음
- 서브점수: "기반점수 A 참여율 B 전환율 C" 텍스트만 (숫자 없음)

### 기대 동작 (목업 기준)
1. 등급 영역 색상: 선명한 빨강(#ef4444) / 노랑(#eab308) / 초록(#22c55e)으로 D/C/B/A 구간 구분
2. 포인터: 호(arc) 위 원형 도트(circle)로 변경
3. 파란 진행 아크 제거
4. 0/50/100 마커 제거
5. 배경 회색 호(#e2e8f0) 추가
6. 서브점수 카드: 기반점수 72(A) / 참여율 55(B) / 전환율 48(C) — 각각 숫자 + 등급 배지가 있는 독립 카드 형태
7. 중앙: 종합점수 숫자 + "B등급" + "7일 기준 · 전체 광고 합산" 텍스트

### 하지 말 것
- 점수 계산 로직(t3-engine) 변경 금지
- 타겟 중복 분석 섹션 변경 금지
- 콘텐츠 탭 변경 금지

## T8: 과거데이터 수동 수집 기능

### 현재 동작
과거데이터 백필 기능 없음. collect-daily는 당일/전일만 수집.

### 기대 동작
1. /admin/protractor 에 "수동 수집" 버튼 추가
2. 계정 선택 드롭다운 + 기간 선택 (7일/30일/90일)
3. 클릭 시 API 호출: /api/admin/backfill
4. 선택된 계정 1개에 대해 지정 기간만큼 일별 순차 수집 (Meta API)
5. 수집 중 진행 상태 표시 ("수집 중... 15/30일")
6. 1계정씩 순차 처리 (동시 수집 금지 — API rate limit 방지)
7. 완료 시 "30일 데이터 수집 완료" 토스트

### 하지 말 것
- 기존 collect-daily 크론 로직 변경 금지
- 여러 계정 동시 수집 금지
- 벤치마크 재계산은 이 TASK 범위 밖

## T9: QA UI AI 말투 제거

### 현재 동작
QA 헬프데스크 전반의 UI 텍스트에 AI스러운 말투 잔존 ("~하실 수 있습니다", "~을 제공합니다" 등).

### 기대 동작
모든 UI 텍스트를 자연스러운 한국어로 교체. 간결하고 직접적인 톤. 예:
- "광고 성과를 분석하실 수 있습니다" → "광고 성과를 확인하세요"
- "데이터를 제공합니다" → "데이터를 보여드려요"
- "설정이 완료되었습니다" → "설정 완료"

### 하지 말 것
- 정보공유 글 생성 프롬프트 변경 금지 (별도 작업)
- 기능 로직 변경 금지 — 텍스트만 수정
