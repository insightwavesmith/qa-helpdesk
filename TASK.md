## 태스크

### T1. 대시보드 "최근 게시글" 데이터 소스 수정 → frontend-dev
- 파일: `src/app/(main)/dashboard/` 관련 컴포넌트
- 의존: 없음
- 현상: KPI에 게시글 18개 표시되지만, "최근 게시글" 섹션은 "등록된 게시글이 없습니다" 출력
- 원인 추정: 이전 `posts` 테이블 참조 또는 카테고리 필터 불일치
- 완료 기준:
  - [ ] "최근 게시글" 섹션에 contents 테이블 published 글 최신 3~5개 표시
  - [ ] KPI 숫자와 일치 (published 글 카운트)
  - [ ] npm run build 성공

### T2. 정보공유 상세 마크다운 렌더링 수정 → frontend-dev
- 파일: `src/app/(main)/posts/[id]/` 관련 컴포넌트
- 의존: 없음
- 현상:
  1. `# 제목` 마크다운 헤더가 텍스트로 출력됨
  2. `<!-- IMG: ... -->` HTML 주석이 그대로 노출
  3. 테이블이 열 구분 없이 텍스트 한 줄로 표시
- 완료 기준:
  - [ ] 마크다운 `#`, `##`, `###` 헤딩이 정상 렌더링
  - [ ] `<!-- ... -->` 주석이 화면에 표시되지 않음
  - [ ] 마크다운 테이블이 표 형태로 렌더링
  - [ ] 기존 글(이미 발행된 18개)이 정상 표시

### T3. HTML 엔티티 디코딩 → frontend-dev
- 파일: 콘텐츠 관리 테이블 컴포넌트 (T2와 같은 담당)
- 의존: 없음
- 현상: `&#8217;` 같은 HTML 엔티티가 디코딩 안 됨 ("Top Facebook Updates You Can&#8217;t Miss")
- 완료 기준:
  - [ ] 제목에서 HTML 엔티티가 정상 문자로 표시
  - [ ] 콘텐츠 관리 + 정보공유 목록 둘 다 확인
