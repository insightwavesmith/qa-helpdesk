# TASK: 수집 UI 정리 + 전체계정 수집 버튼

CLAUDE.md 읽고 delegate 모드로 팀원 만들어서 진행해라.

---

## 목표
관리자 수집 UI를 정리하고, "전체계정 수집" 버튼 하나로 광고+믹스패널+타겟중복을 한번에 수집한다.

## 수정사항

### 1. 위쪽 "일괄 데이터 수집" 섹션 제거
- T3(`c35b053`)에서 추가한 체크박스 선택 UI, "전체 선택", "선택 해제", "선택 수집(0)" 버튼 등 모두 제거
- 위쪽 섹션 전체를 삭제

### 2. 기존 UI에 "전체계정 수집" 버튼 추가
- 기존 **광고계정 선택 드롭다운 + 기간 선택** UI 옆에 "전체계정 수집" 버튼 추가
- 기간은 기존 기간 선택 컨트롤 값을 공유 (관리자가 기간 선택 → 전체계정 수집 클릭)
- 버튼 클릭 시: 선택된 기간으로 **모든 active 광고계정** 수집 실행

### 3. 수집 범위 — 한번에 실행
"전체계정 수집" 클릭 시 순차적으로:
1. **광고 성과 수집** (daily_ad_insights) — 기존 collect-daily 로직
2. **믹스패널 매출 수집** (daily_lp_metrics) — 기존 collect-mixpanel 로직
3. **타겟중복률** — 기존 overlap API 로직 (아마 데일리 수집에 포함되어 있을 수 있으니 확인)

수집 진행 상태를 SSE 스트리밍으로 보여주면 좋지만, 최소한 프로그레스 표시 (X/Y 계정 수집 중)

### 4. 개별 계정 수집은 유지
기존 광고계정 선택 → 기간 선택 → 수집 기능은 그대로 유지. "전체계정 수집"은 추가 버튼.

## 주의사항
- 기존 API 엔드포인트 재활용 (`/api/protractor/collect-daily`, `/api/protractor/collect-mixpanel`)
- `accountIds: "all"` 파라미터로 전체 계정 수집 가능해야 함
- Vercel 함수 타임아웃 고려 (Pro: 300s) — 계정 많으면 병렬 또는 chunking
