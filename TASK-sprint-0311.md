## T1. QA 답변에서 참고 강의자료 소스 숨기기

### 고객 시나리오
수강생이 QA에서 질문 → AI 답변을 받는데, 답변 안에 "참고: [강의자료명]" 같은 소스 정보가 노출되고 있다. 수강생 입장에서 이건 내부 정보이므로 보이면 안 됨.

### 기대 동작
- QA 답변이 표시될 때 참고한 지식베이스/강의자료 소스 정보가 **고객(수강생)에게는 보이지 않아야** 함
- 관리자에게는 여전히 보여도 됨 (디버깅용)
- AI 답변 생성 시 소스를 참고하는 건 유지, 단지 **프론트에서 렌더링할 때 숨기기**

### 힌트
- `/api/qa-chatbot` 응답에 sources/references 필드가 있을 수 있음
- 프론트에서 해당 필드를 렌더링하는 부분 찾아서 수강생 role일 때 숨기기
- 또는 API 응답 자체에서 role 기반 필터링

## T2. 답변 수정 기능 추가

### 고객 시나리오
관리자/조교가 QA 질문에 답변을 달았는데, 오타가 있거나 내용을 보완하고 싶다. 현재는 수정 기능이 없어서 삭제 후 재작성해야 함.

### 기대 동작
- 답변 작성자(관리자/조교)가 **이미 게시된 답변을 수정**할 수 있어야 함
- 수정 버튼 → 편집 모드 → 저장
- 수정 이력은 선택사항 (없어도 됨)
- 수강생이 자기 질문은 수정 가능해도 됨 (선택)

### 힌트
- `answers` 테이블에 이미 데이터 있음
- 프론트: 답변 카드에 "수정" 버튼 (본인 답변만)
- API: PUT `/api/questions/[id]/answers/[answerId]`
- RLS: 작성자 본인만 수정 가능

## T3. 총가치각도기 전체 계정 일괄 수집 + 선택 수집

### 고객 시나리오
관리자가 수강생들의 광고 성과를 한번에 확인하고 싶은데, 현재는 계정별로 따로 수집해야 한다. "전체 수집" 버튼으로 등록된 모든 광고계정 데이터를 일괄 수집하거나, 원하는 계정만 선택해서 수집하고 싶다.

### 기대 동작
1. 관리자 페이지(광고계정 관리 또는 총가치각도기 관리)에 **"전체 수집" 버튼** 추가
   - 클릭하면 `ad_accounts` 테이블의 모든 활성 계정에 대해 collect-daily 실행
2. **계정 선택 수집** — 체크박스로 원하는 계정만 골라서 수집 트리거
3. 수집 진행 상태 표시 (로딩/완료/에러)
4. **캠페인 범위 확인**: 현재 collect-daily가 판매(OUTCOME_SALES) 캠페인만 수집하는지, 전체 캠페인(잠재고객 포함)을 수집하는지 확인하고 → **전체 캠페인 수집으로 확장** (objective 필터가 있으면 제거)

### 힌트
- 기존 `/api/protractor/collect` 또는 `/api/cron/collect-daily` 코드 참고
- 새 API: POST `/api/admin/protractor/collect` — body에 `{ accountIds: string[] | "all" }` 
- 프론트: 관리자 광고계정 관리 페이지에 수집 UI 추가
- Meta Marketing API 호출 시 campaign objective 필터 확인 → 필터 있으면 제거해서 전체 캠페인 수집
