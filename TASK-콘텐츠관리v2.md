## T1. 아카이브 기능

### 이게 뭔지
관리자가 콘텐츠를 "아카이브"하면 목록에서 사라지되 DB에는 남는다.

### 왜 필요한지
콘텐츠를 참조하는 FK 테이블이 6개(distributions, email_logs, knowledge_chunks, knowledge_usage, content_relations×2) 있어서 하드 삭제가 실패한다. 아카이브로 소프트 삭제가 필요하다.

### 파일
- 콘텐츠 상세 설정 UI (삭제 버튼 옆에 "아카이브" 버튼 추가)
- `src/actions/contents.ts` (아카이브 액션)

### 검증 기준
- "아카이브" 클릭 → status가 "archived"로 변경
- 아카이브된 콘텐츠는 콘텐츠 탭 기본 목록에서 안 보임
- (선택) 필터에서 "아카이브됨" 선택하면 아카이브된 항목 확인 가능

### 하지 말 것
- DB 스키마 변경 금지 (archived status는 이미 존재하고 조회 시 기본 제외됨)

## T2. 삭제 기능 수정

### 이게 뭔지
콘텐츠 삭제 시 연관 데이터(FK 참조)를 먼저 정리한 후 삭제한다.

### 왜 필요한지
현재 deleteContent()가 FK 위반으로 실패한다.

### 파일
- `src/actions/contents.ts` — deleteContent() 함수

### 검증 기준
- 삭제 클릭 → 확인 다이얼로그 → 연관 데이터 정리 → 콘텐츠 삭제 성공
- 삭제 후 목록에서 즉시 사라짐

### 하지 말 것
- FK 제약조건 자체를 CASCADE로 변경하지 마라 (다른 기능에 영향)

## T3. 콘텐츠 탭에 공지사항 통합

### 이게 뭔지
관리자가 콘텐츠 탭에서 공지사항을 생성하고 관리할 수 있게 한다.

### 왜 필요한지
현재 공지사항 전용 생성 페이지가 없다. 콘텐츠 탭의 "새 콘텐츠" 버튼으로 공지를 만들고, 발행하면 수강생 공지사항 페이지에 표시되어야 한다.

### 파일
- 콘텐츠 탭 목록 필터 (info_share + notice + manual 표시)
- "새 콘텐츠" 생성 폼 (카테고리에 "공지" 선택지 추가)
- 수강생 공지사항 페이지 (/notices) — 발행된 공지 표시

### 검증 기준
- 콘텐츠 탭에서 info_share와 공지(notice) 항목 모두 표시
- "새 콘텐츠" → 카테고리 "공지" 선택 → 작성 → 발행
- 발행된 공지가 수강생 /notices 페이지에 최신순으로 표시
- 큐레이션 원본(crawl, youtube, blueprint, lecture)은 여전히 콘텐츠 탭에 안 보임

### 하지 말 것
- 큐레이션 탭 건드리지 마라
- 기존 정보공유 생성 흐름 변경하지 마라
