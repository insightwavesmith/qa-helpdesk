# Q&A UX 리디자인 계획서

## 1. 개요
- Q&A 페이지의 목적을 명확히 분리: "지식베이스 열람" vs "내 질문 관리"
- 수강생이 Q&A에 들어왔을 때 답변된 양질의 콘텐츠만 보이도록 변경

## 2. 핵심 요구사항

### 기능적 요구사항
- FR-01: Q&A 메인(전체 Q&A)에는 답변완료(status=answered) 질문만 표시
- FR-02: "내 질문" 탭 추가 — 로그인한 사용자의 질문만 표시 (모든 상태)
- FR-03: Q&A 메인에서 미답변/답변완료 상태 필터 제거, 카테고리 필터만 유지
- FR-04: "내 질문" 탭에서 상태 배지(답변 대기/답변 완료) 표시
- FR-05: "질문하기" 버튼은 양쪽 탭 모두 노출
- FR-06: 정보공유 탭 → 공지만 남기기 (정보/웨비나 카테고리 제거)

### 비기능적 요구사항
- NFR-01: 탭 전환 시 페이지 새로고침 없이 클라이언트에서 처리
- NFR-02: 모바일 반응형 유지

## 3. 용어 정의
| 용어 | 영문 | 설명 |
|------|------|------|
| 전체 Q&A | All Q&A | 답변완료된 질문 아카이브 (지식베이스) |
| 내 질문 | My Questions | 본인이 작성한 질문 목록 |
| 답변 대기 | open | AI 답변 생성 또는 관리자 승인 대기 |
| 답변 완료 | answered | 관리자 승인된 답변이 있는 질문 |

## 4. 범위
- 포함: Q&A 페이지 탭 구조 변경, 필터 수정, 정보공유 카테고리 정리
- 제외: 관리자 페이지 변경, AI 답변 로직 변경, 벡터DB 재임베딩

## 5. 성공 기준
- [ ] Q&A 메인에 미답변 질문이 노출되지 않음
- [ ] 본인 질문은 "내 질문" 탭에서 상태와 함께 확인 가능
- [ ] 정보공유 탭에 공지만 표시
- [ ] 모바일에서 탭 전환 정상 동작

## 6. 현재 상태 (as-is)
- Q&A 페이지: 전체/미답변/답변완료 필터 + 카테고리 필터
- 모든 질문이 상태 무관하게 노출
- 정보공유: 정보/공지/웨비나 3개 카테고리

## 7. 목표 상태 (to-be)
- Q&A 페이지: [전체 Q&A | 내 질문] 탭 + 카테고리 필터(전체 Q&A에서만)
- 전체 Q&A: answered 상태만
- 내 질문: 본인 질문 전체 (상태 배지 포함)
- 정보공유: 공지만
