# TASK: 사전계산 후 로드 — 적용 가능 영역 전수 조사

## 목적
현재 페이지 로드 시 실시간 계산하는 영역들을 찾아서, 사전계산(수집/크론 시점에 미리 계산하여 DB 저장) 후 프론트에서 로드만 하는 구조로 바꿀 수 있는 영역을 전부 파악.

## 요구사항

### 1. 현재 코드 전수 조사
프로젝트 전체에서 다음 패턴을 찾아라:
- 페이지 로드 시 DB 조회 → 계산 → 표시하는 영역
- API route에서 복잡한 집계/계산을 실시간으로 하는 영역
- 관리자 페이지와 수강생 페이지 모두 포함

### 2. 영역별 분석
각 영역에 대해:
- **현재 동작**: 어떤 데이터를 어떻게 계산하는지
- **사전계산 가능 여부**: 수집/크론 시점에 미리 계산 가능한지
- **효과**: 로딩 속도 개선 정도 (데이터 규모, 계산 복잡도)
- **구현 난이도**: 쉬움/보통/어려움
- **영향 범위**: 관리자용/수강생용/공용

### 3. 대상 범위
- 총가치각도기 (T3 엔진 계산)
- 벤치마크 통계
- 경쟁사 분석 집계
- 콘텐츠 랭킹/정렬
- QA 통계
- 대시보드 요약 데이터
- 기타 발견되는 영역

## 산출물
조사 결과를 `docs/precompute-audit.md`에 정리:
- 영역 목록 (관리자/수강생 구분)
- 우선순위 (효과 × 난이도)
- 추천 구현 순서

## 주의
- 코드 수정은 하지 마. 조사+보고만.
- `npm run build` 깨뜨리지 마.
