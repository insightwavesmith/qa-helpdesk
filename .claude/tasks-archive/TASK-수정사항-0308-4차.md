## T2. 초대코드 사용량 소진 안 됨 — 반복 수정 요청

가입은 되는데 초대코드 사용량이 줄지 않는다. bs-06 코드로 가입했는데 0/50 그대로.

**이것도 end-to-end 추적해라:**
1. 가입 시 초대코드 검증하는 코드 위치
2. 검증 후 사용량 차감하는 코드가 있는지
3. 있다면 왜 실행 안 되는지, 없다면 추가

### 파일

- `src/actions/` 관련 파일
- DB: invitation_codes 테이블

- 초대코드로 가입 성공 → 사용량 0/50 변동 없음

### 기대 동작
- 가입 성공 시 사용량 +1 (0/50 → 1/50)

## T4. 키워드 검색 로직 추가 (경쟁사 분석기)

현재 검색이 브랜드명(page_search)만 지원한다. 한글 브랜드명(젝시미스 등)은 Meta에서 인식 못 해서 결과가 안 나온다.

3가지 검색 강화:

### 4-1. 폴백 검색
- page_search 결과 0건이면 → ad_library 키워드 검색(q=입력값)으로 자동 전환
- 광고 본문/제목에 검색어가 포함된 광고를 찾는다

### 4-2. 브랜드 사전
- 한글↔영문 매핑 테이블 (DB 또는 코드 상수)
- "젝시미스" → "xexymix", "올리브영" → "oliveyoung", "무신사" → "musinsa" 등 주요 20~30개
- page_search 전에 사전 매칭 시도 → 영문명으로 검색

### 4-3. 자동 영문 변환
- 사전에 없는 브랜드 → 구글 자동완성 또는 검색 API로 영문명 추정
- 비용 없는 방법 우선 (구글 Suggest API 등)

### 파일
- `src/lib/meta-ad-library.ts`
- `src/app/api/competitor/search/route.ts`
- `src/app/(main)/protractor/competitor/` 관련 파일

### 기대 동작
- "젝시미스" 검색 → 사전 매칭 "xexymix" → 15건 결과
- "아디다스" 검색 → 사전 매칭 "adidas" → 결과 표시
- 사전에 없는 한글 → 폴백 검색 → 광고 본문에서 찾기

### 하지 말 것
- 기존 브랜드 검색(page_search) 로직을 삭제하지 마라 (추가만)
- 모니터링 기능 건드리지 마라
