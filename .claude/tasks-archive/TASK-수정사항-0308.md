## 목표
서비스 오픈(3/9) 전에 수강생이 실제로 사용할 때 막히는 6가지를 해결한다.

## 빌드/테스트
- npm run build 성공 필수
- 테스트 계정: smith@test.com / test1234! (admin), student@test.com / test1234! (student)

## T1. 수강생이 올린 Q&A 질문 삭제 불가

수강생이 Q&A에 질문을 올린 후 잘못 올렸거나 수정하고 싶을 때 삭제할 수 없다.
자기가 올린 글은 자기가 삭제할 수 있어야 한다.

### 파일
- `src/app/(main)/questions/[id]/page.tsx`
- `src/actions/questions.ts`

### 현재 동작
- 수강생이 올린 질문에 삭제 버튼 없음

### 기대 동작
- 자기가 올린 질문에만 삭제 버튼 표시
- 삭제 클릭 → 확인 모달 → 삭제 완료
- 관리자는 모든 질문 삭제 가능 (기존 유지)

### 하지 말 것
- 다른 사람의 질문을 삭제할 수 있게 하지 마라
- AI 답변 생성 로직 건드리지 마라

내가(관리자) 올린 수강후기를 삭제할 수 없다.
올린 사람은 자기 글을 삭제할 수 있어야 한다.

### 파일

- `src/actions/` 관련 파일

### 현재 동작

- 관리자는 모든 후기 삭제 가능

### 하지 말 것

## T3. 총가치각도기 해석 가이드 삭제 + 중복률 UI 변경

총가치각도기에서 "해석 가이드" 텍스트가 노출되고 있다. 이건 내부 계산 방식인데 수강생한테 보여주면 안 된다.
그리고 원형차트 옆에 숫자(32.4%)가 나와있는데, 이것 대신 광고세트별 요약을 붙이고 세트조합 중복률을 바차트로 보여줘야 한다.

### 파일
- `src/app/(main)/protractor/` 관련 컴포넌트

### 현재 동작
- 해석 가이드 텍스트 노출 (전체 중복률 공식, 60%/30% 기준 설명)
- 원형차트 오른쪽에 숫자(32.4%) 표시

### 기대 동작
- 해석 가이드 텍스트 전부 삭제 (공식, 기준 설명 모두)
- 숫자(32.4% 같은) 삭제
- 원형차트 오른쪽에 **광고세트별 요약** 표시
- 그 아래에 **세트조합 중복률 바차트** 표시
- 계산 방식은 절대 노출하지 않는다

### 하지 말 것
- 중복률 계산 로직 자체를 변경하지 마라 (표시만 바꿔라)
- 성과 요약 탭 건드리지 마라

## T4. 개인정보처리방침 링크 클릭 시 페이지 안 나옴

회원가입 화면에서 "개인정보처리방침에 동의합니다" 링크를 클릭하면 개인정보처리방침 페이지가 열려야 한다.
지금은 클릭해도 페이지가 안 나온다.

### 파일
- `src/app/(auth)/signup/page.tsx`

### 현재 동작
- "개인정보처리방침" 링크 클릭 → 페이지 안 열림

### 기대 동작
- 링크 클릭 → `/privacy` 페이지가 새 탭으로 열림
- `/privacy` 페이지에 개인정보처리방침 내용 정상 표시

### 하지 말 것
- 회원가입 플로우 변경하지 마라
- 체크박스 로직 건드리지 마라

## T5. 경쟁사 분석기 "더보기" 클릭 시 다음 광고 안 나옴

경쟁사 분석기에서 검색 후 "더보기 (30/54)" 버튼을 누르면 다음 페이지 광고가 나와야 한다.
지금은 클릭해도 추가 로드가 안 된다.

### 파일
- `src/app/(main)/protractor/competitor/` 관련 파일

### 현재 동작
- "더보기" 버튼 클릭 → 추가 광고 로드 안 됨

### 기대 동작
- "더보기" 클릭 → 다음 페이지 광고 카드 추가 표시
- 전체 54건이면 30건 이후 나머지 24건 로드

### 하지 말 것
- 검색 드롭다운 UI 건드리지 마라
- 필터/정렬 로직 건드리지 마라

## T6. 경쟁사 분석기 브랜드 모니터링 등록(핀) 안 됨

검색 결과에서 브랜드 오른쪽 핀 버튼을 누르면 모니터링에 등록되어야 한다.
지금은 핀을 눌러도 등록이 안 된다.

### 파일
- `src/app/(main)/protractor/competitor/` 관련 파일

### 현재 동작
- 브랜드 검색 → 핀 버튼 클릭 → 모니터링에 등록 안 됨

### 기대 동작
- 핀 클릭 → 모니터링 패널에 브랜드 추가
- 이미 등록된 브랜드는 핀 비활성화 또는 "등록됨" 표시

### 하지 말 것
- 모니터링 패널의 기존 기능 건드리지 마라
- 검색 드롭다운 건드리지 마라
