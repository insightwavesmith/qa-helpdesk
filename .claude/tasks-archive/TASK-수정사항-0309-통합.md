# TASK: 0309 통합 수정 (2건)

## T1. 더보기 페이지네이션 — A안: 서버에서 중복 제거 후 표시

현재 SearchAPI.io에서 60건 가져오면 30건 보여주고 더보기 누르면 중복 때문에 1건만 나온다.

### 변경 방향
서버(API route)에서 중복을 먼저 제거하고, 중복 제거된 실제 건수를 프론트에 보여준다.

### 구체적 수정
1. API route (`search/route.ts`)에서 SearchAPI.io 응답 받을 때 ad_archive_id 기준으로 중복 제거
2. 프론트에 전달하는 total_count는 중복 제거 후 실제 건수
3. "총 47건" (중복 제거 후) 표시 → 30건 보여주고 → 더보기 → 나머지 17건
4. 프론트의 중복 제거 로직 제거 (서버에서 이미 처리)
5. "중복 광고가 많습니다" 토스트 메시지 제거

### 하지 말 것
- 검색/필터 건드리지 마라
- 핀/모니터링 건드리지 마라

모니터링에 등록된 브랜드 아래에 @인스타아이디가 표시되어야 한다.

### 수정
1. 모니터링 등록(핀) 시 ig_username도 함께 DB에 저장
2. 모니터링 패널에서 브랜드명 아래에 @ig_username 표시
