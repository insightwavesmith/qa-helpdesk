# TASK: 경쟁사 분석기 버그 수정 3건

## 타입
개발

## T1: 광고 카드에 랜딩페이지 버튼 추가
**이게 뭔지**: 경쟁사 분석기에서 광고 노출할 때 "소재 보기", "다운로드" 버튼 옆에 "랜딩페이지" 버튼도 추가
**왜 필요한지**: 수강생이 경쟁사 광고의 랜딩페이지를 바로 확인하고 싶은데 현재 버튼이 없음
**구현 내용**:
- 광고 카드 하단 버튼 영역에 "랜딩페이지" 버튼 추가
- 외부 링크 아이콘 + "랜딩페이지" 텍스트
- 클릭 시 광고의 landing_page_url을 새 탭으로 열기
- 랜딩페이지 URL이 없는 광고는 버튼 숨김
- 스타일: 기존 버튼(소재보기/다운로드)과 동일한 라인

**관련 파일**: `src/components/protractor/competitor/` 내 광고 카드 컴포넌트

## T2: 키워드 검색 → 핀 등록 시 인스타 @계정 미표시
**이게 뭔지**: 키워드 검색으로 광고 찾아서 핀(모니터링 등록) 하면 모니터링 목록에서 인스타 계정(@...)이 안 뜸
**왜 필요한지**: 브랜드 검색으로 핀하면 @계정이 뜨는데, 키워드 검색에서 핀하면 안 뜸 — 어디서 등록했는지에 따라 동작이 달라지면 안 됨
**구현 내용**:
- 키워드 검색 결과에서 핀 등록 시 ig_username(인스타 계정)도 함께 저장하도록 수정
- 모니터링 목록에서 @계정 표시 로직 확인 — 데이터가 없어서 안 뜨는 건지, 표시 로직 문제인지 파악
- SearchAPI.io 키워드 검색 응답에 ig_username 포함 여부 확인

**관련 파일**: `src/components/protractor/competitor/` 내 핀 관련 컴포넌트, `src/app/api/protractor/` 내 모니터링 API

## T3: 더보기 페이지네이션 여전히 안 됨
**이게 뭔지**: 더보기(30/3,527) 버튼 누르면 "검색 API 호출 실패: Either `q` or `page_id` or `location_id` must be present" 에러
**왜 필요한지**: 첫 페이지(30건)만 보이고 나머지 3,497건을 볼 수 없음 — 핵심 기능
**구현 내용**:
- 커밋 `1484ca8`에서 "page_token만 단독 전송"으로 수정했는데 여전히 에러 발생
- 에러 메시지가 `q` or `page_id` 필수라고 하니까, page_token 전송 시 q나 page_id가 누락되고 있을 가능성
- 실제 API 호출 시 어떤 파라미터가 전송되는지 확인
- SearchAPI.io의 page_token이 검색 조건을 이미 인코딩하고 있다면 page_token만 보내야 하고, 아니면 기존 검색 조건도 같이 보내야 함
- 프론트엔드에서 더보기 클릭 시 어떤 API를 어떤 파라미터로 호출하는지 추적

**관련 파일**: `src/lib/protractor/meta-ad-library.ts`, `src/app/api/protractor/competitor/route.ts`, `src/components/protractor/competitor/competitor-dashboard.tsx`
