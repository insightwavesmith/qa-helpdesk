# Ouroboros Interview — 요구사항 결정화

> 출처: Q00/ouroboros. "Stop prompting. Start specifying."

## 프로세스

### 1. Interview (소크라테스식 질문)
- 모호한 요구사항에 대해 질문으로 가정을 노출
- "누가 쓰는가?", "안 되면 어떻게 되는가?", "이미 있는 건?"
- 모호성 점수가 0.2 이하가 될 때까지 반복
- 한 번에 질문 1개만

### 2. Seed (명세 결정화)
인터뷰 완료 후 산출물:
```
- 문제 정의 (1줄)
- 사용자 (누구)
- 핵심 요구사항 (3-5개)
- 안 하는 것 (명시적 범위 제한)
- 가정 (확인 안 된 것)
- 성공 기준 (검증 가능한 조건)
```

### 3. Execute → Evaluate → 반복
Seed 기반으로 실행 → 평가 → Seed 업데이트 → 재실행

## 규칙
- 추측 금지. 모르면 질문
- "상황에 따라" 단독 답변 금지
- 가정은 반드시 명시
