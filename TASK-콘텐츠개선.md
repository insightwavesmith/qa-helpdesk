# TASK: 콘텐츠 시스템 버그 수정 + 구조 개선

## 배경
관리자가 콘텐츠(공지, 고객사례 등)를 발행해도 수강생에게 안 보이고,
콘텐츠 수정 시 썸네일/이미지 변경이 안 되고,
"콘텐츠"와 "정보공유"가 별개 탭으로 존재해서 혼란스럽다.

## T1: 과거 데이터 수집 시 총가치각도기 점수가 0으로 나옴

### 이게 뭔지
수강생이 과거 광고 데이터를 수집하면 "벤치마크 데이터가 없어 점수를 계산할 수 없습니다" 경고가 뜨고, 기반점수·참여율·전환율 전부 0점 C등급으로 표시된다.

### 왜 필요한지
총가치각도기의 핵심이 점수인데 점수가 안 나오면 수강생한테 의미가 없다.

### 구현 내용
- `src/app/api/protractor/total-value/route.ts` 의 `fetchBenchmarks()` 디버깅
- 벤치마크 테이블에 180건이 이미 있다. `creative_type` 매칭 로직 확인
  - `dominantCT` (수강생의 가장 많은 소재 타입)이 벤치마크에 없을 때 fallback이 `ALL`인데, ALL도 못 찾는지 확인
  - `calculated_at` 날짜 필터가 과거 데이터와 안 맞는지 확인
  - `ranking_group = 'ABOVE_AVERAGE'` 조건이 맞는지 확인
- 벤치마크 매칭 실패 시 로그 추가해서 원인 특정
- 최소한 ALL fallback은 항상 점수가 나와야 함

## T2: 관리자가 발행한 콘텐츠가 수강생 화면에 안 보임

### 이게 뭔지
관리자가 "콘텐츠 관리"에서 공지·고객사례 등을 만들어서 "발행"해도, 수강생의 "정보공유" 탭에 나타나지 않는다.

### 왜 필요한지
관리자가 만든 콘텐츠가 관리자 콘텐츠 탭에서도 안 보이면 관리가 불가능하다.

### 구현 내용
- **원인**: `src/app/(main)/admin/content/page.tsx`의 SWR 쿼리가 `sourceType: "info_share"`로 하드코딩
- 공지(notice), 고객사례(case_study) 등 직접 만든 콘텐츠는 `source_type`이 다르기 때문에 필터에서 걸러짐
- 수정: "콘텐츠" 탭에서 `sourceType` 필터를 제거하거나, 모든 source_type을 보여주도록 변경
- 또는: 콘텐츠 탭에 source_type 필터 드롭다운 추가 (전체/info_share/manual/notice 등)

## T3: 콘텐츠 수정 시 썸네일·이미지 변경 불가

### 이게 뭔지
발행된 콘텐츠를 수정할 때:
1. 썸네일 이미지를 변경하는 UI가 안 먹힘
2. 본문 에디터에서 이미지 추가 시 파일 선택/URL 입력이 작동하지 않음 (이미지_02 스크린샷)

### 왜 필요한지
한번 올린 콘텐츠를 수정할 수 없으면 삭제 후 재작성해야 한다. 운영이 불가능하다.

### 구현 내용
- 썸네일: `src/components/content/detail-sidebar.tsx` → `handleThumbnailUpload()`
  - Supabase Storage `content-images` 버킷 권한(RLS) 확인
  - `<input type="file">` ref 동작 확인
- 에디터 이미지: `src/components/content/mdx-editor-wrapper.tsx` → `imageUploadHandler()`
  - MDXEditor의 InsertImage 다이얼로그 → 파일 선택 시 `imageUploadHandler` 호출되는지
  - Storage 업로드 에러 확인 (CORS, 버킷 정책, 파일 크기 제한)
  - 에디터 수정 모드에서 이미지 플러그인이 정상 로드되는지
- 수정 모드 진입 시 에디터 초기화 문제 확인 (기존 markdown 로드 → 수정 → 이미지 추가)

## T4: 콘텐츠 탭 / 정보공유 탭 통합

### 이게 뭔지
현재 수강생에게 "정보공유"(+ "공지사항" 사이드바)가 별도로 존재하고, 관리자에겐 "콘텐츠 관리"가 있다.
수강생 입장에서 같은 글이 어디에 있는지 헷갈린다.

### 왜 필요한지
하나의 콘텐츠 시스템이 두 개 탭으로 나뉘어 있으면 혼란스럽다. 정보공유 탭 하나로 통합해야 한다.

### 구현 내용
- 수강생 네비게이션에서 "공지사항"(사이드바 `/notices`)을 제거하고, 공지도 정보공유 탭 안에서 카테고리 필터로 표시
- 또는: 정보공유 탭 상단에 카테고리 탭 (전체 / 공지 / 강의 / 고객사례 등)
- `/notices` 라우트 → `/posts?category=notice` 로 리다이렉트
- 수강생 헤더(`student-header.tsx`) + 사이드바(`app-sidebar.tsx`) + 모바일 탭바(`mobile-tab-bar.tsx`) 수정
- T2 수정 후에 진행 (T2에서 콘텐츠가 정보공유에 나오는 게 선행)

## 산출물
- T1: total-value API 수정 + 점수 정상 표시 확인
- T2: 발행 콘텐츠 → 수강생 정보공유 노출 확인
- T3: 썸네일 변경 + 에디터 이미지 추가 정상 작동
- T4: 네비게이션 통합 + 공지사항 카테고리 필터
