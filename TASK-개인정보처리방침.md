# TASK: 회원가입 개인정보처리방침 필수동의

## 목표
수강생이 회원가입할 때 개인정보처리방침에 필수 동의해야 가입이 진행된다.
동의 없이는 가입 버튼이 비활성화되어야 하고, 개인정보처리방침 내용은 우리 서비스가 실제로 수집·이용하는 데이터를 정확히 반영해야 한다.

## 빌드/테스트
- npm run build 성공 필수
- 회원가입 플로우 전체 테스트

## T1. 회원가입 페이지에 필수동의 체크박스 추가

### 고객 시나리오
수강생이 회원가입 폼을 작성하고 "가입" 버튼을 누르기 전에 개인정보처리방침 동의 체크박스가 보인다.
체크하지 않으면 가입 버튼이 비활성화(disabled)되어야 한다.
"개인정보처리방침" 텍스트를 클릭하면 내용을 볼 수 있어야 한다 (새 탭 또는 모달).

### 기대 동작
- 체크박스: "개인정보처리방침에 동의합니다 (필수)" — 체크 전에는 가입 버튼 disabled
- "개인정보처리방침" 텍스트 → `/privacy` 링크 (새 탭)
- 체크 후 가입 → 정상 진행

## T2. 개인정보처리방침 페이지 내용 보강

### 고객 시나리오
수강생이 동의하기 전에 어떤 데이터를 수집하는지 읽어볼 수 있어야 한다.
현재 `/privacy` 페이지가 있지만 Meta 앱 등록용으로 간략하게 작성되어 있다.
우리 서비스 기준으로 내용을 보강해야 한다.

### 포함할 내용
- 수집하는 개인정보: 이메일, 이름, 비밀번호
- 이용 목적: 서비스 제공, 수강 관리, 콘텐츠 추천
- 보유 기간: 회원 탈퇴 시까지
- 제3자 제공: Meta 광고 라이브러리 API 연동 (SearchAPI.io 경유, 개인정보 전송 없음)
- 수집하는 이용 데이터: 검색 기록, 콘텐츠 열람 기록, 광고 분석 기록 (서비스 개선 목적)
- 동의 철회: 회원 탈퇴로 가능
- 문의: 서비스 관리자 연락처

### 파일
- `/privacy` 페이지 (경로 확인 후 수정)

## T3. 동의 기록 저장

### 고객 시나리오
법적으로 사용자가 언제 동의했는지 기록이 남아야 한다.

### 기대 동작
- 회원가입 시 동의 시점(timestamp) DB에 저장
- 기존 users 테이블에 `privacy_agreed_at TIMESTAMPTZ` 컬럼 추가
- 또는 별도 테이블 `user_consents(user_id, consent_type, agreed_at)`

## 하지 말 것
- 기존 회원가입 플로우 깨지지 않게 할 것
- 로그인 페이지 건드리지 마라
- 이미 가입한 사용자한테 영향 없어야 함 (기존 유저는 NULL 허용)

## 검증 기준
- 체크 안 하고 가입 → 버튼 disabled
- 체크하고 가입 → 정상 가입
- /privacy 페이지 → 보강된 내용 표시
- DB에 privacy_agreed_at 기록됨
