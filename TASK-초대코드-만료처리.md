# TASK: 초대코드 만료 자동 비활성화

## 목표
초대코드의 만료일이 지났는데 "활성" 상태로 남아있다. 
만료일이 지난 초대코드는 자동으로 비활성 처리되어야 한다.
수강생이 만료된 코드로 가입하면 안 된다.

## 빌드/테스트
- npm run build 성공 필수
- 테스트 계정: smith@test.com / test1234! (admin)

## T1. 만료된 초대코드 자동 비활성화

### 고객 시나리오
관리자가 초대코드를 만들면서 만료일을 설정한다. 만료일이 지나면 그 코드는 더 이상 사용할 수 없어야 한다.
현재는 만료일이 지나도 "활성" 뱃지가 그대로 남아있어서 관리자가 헷갈린다.
수강생이 만료된 코드로 가입을 시도하면 "만료된 코드입니다" 안내가 나와야 한다.

### 기대 동작
- 초대코드 목록에서 만료일 < 오늘 → 상태 표시를 "만료" (회색 뱃지)로 변경
- 만료된 코드로 회원가입 시도 → "만료된 초대코드입니다" 에러 메시지
- 만료 체크는 실시간 (DB 상태 변경이 아닌 조회 시 만료일 비교로 처리해도 됨)
- 관리자가 만료된 코드를 수동으로 재활성화할 수 있어야 함 (만료일 연장)

## 하지 말 것
- 기존 초대코드 데이터 삭제하지 마라
- 회원가입 플로우 자체를 깨지 않게 할 것
- 활성 상태인 유효한 코드 영향 없어야 함

## 검증 기준
- 만료일 지난 초대코드 → 목록에서 "만료" 표시
- 만료된 코드로 가입 → 에러 메시지
- 유효한 코드로 가입 → 정상 동작
- 관리자가 만료일 연장 → 재활성화
