## T1. 브랜드 검색 → 프로필 클릭 시 광고 안 나옴

수강생이 "올리브영"을 검색하면 드롭다운에 브랜드 목록이 뜬다. 클릭하면 그 브랜드의 광고가 나와야 하는데 빈 화면이다.
검색까지 잘 되는데 결과를 볼 수 없으면 기능이 없는 거나 마찬가지.

### 파일
- `src/app/(main)/protractor/competitor/competitor-dashboard.tsx`
- `src/app/(main)/protractor/competitor/components/brand-search-bar.tsx`

### 현재 동작
- 브랜드 검색 드롭다운 → 정상 (프로필+인스타+좋아요 표시)
- 브랜드 클릭 → 빈 화면, 광고 로드 안 됨

### 기대 동작
- 브랜드 클릭 → 해당 page_id로 광고 검색 → 광고 카드 목록 표시
- 로딩 스피너 → 결과

### 하지 말 것
- 검색 드롭다운 UI 건드리지 마라
- 모니터링 패널 건드리지 마라

## T2. QA 답변에서 볼드(**) 제거

수강생 질문에 AI가 자동 답변을 생성하는데, `**볼드**` 마크다운이 들어간다. 실무 선배가 답변하는 톤인데 볼드 강조는 블로그 글처럼 보여서 어울리지 않는다.

### 파일
- `src/lib/knowledge.ts` — `QA_SYSTEM_PROMPT` 내 마크다운 포맷팅 규칙

### 현재 동작
- 프롬프트에 "**굵기 강조**만 허용" → 답변에 `**총가치 = 입찰가 × 추산행동률 + 광고품질**` 같은 볼드 포함

### 기대 동작
- 프롬프트에서 "굵기 강조만 허용" → "굵기 강조(**) 금지. 평문으로만 작성"으로 변경
- 볼드 없이 자연스러운 평문 답변

### 하지 말 것
- QA 답변 모델 변경하지 마라 (프롬프트만 수정)
- 정보공유 생성 AI 모델(Opus) 변경 금지
- 다른 프롬프트 내용 건드리지 마라 (볼드 규칙만 수정)

## T3. 클라이언트 캐싱 (SWR 도입)

수강생이 페이지를 이동할 때마다 데이터를 새로 불러온다. 대시보드 → 질문 → 대시보드로 돌아오면 또 로딩.
한번 불러온 데이터는 캐시해서 페이지 전환이 즉시 되어야 자연스럽다.

### 파일
- 주요 페이지 컴포넌트들 (대시보드, 질문 목록, 커리큘럼 등)
- 이미 plan 문서 작성됨: `docs/01-plan/features/` 참고

### 현재 동작
- 페이지 전환마다 useEffect + fetch로 데이터 재로딩
- 매번 로딩 스피너 표시

### 기대 동작
- SWR 도입으로 클라이언트 캐싱
- 페이지 전환 시 캐시된 데이터 즉시 표시 → 백그라운드 재검증
- 사용자 체감: 페이지 전환이 즉시

### 하지 말 것
- Server Actions/Server Components 구조 변경하지 마라
- API 라우트 변경하지 마라
