## T1. 진단 색상 판정 기준 버그 수정

### 이게 뭔지
전환율 섹션에서 구매전환율 7.31% (벤치마크 7.72%)가 빨간색으로 표시됨. 94.7% 달성이면 노란색(주의)이어야 맞음.

### 왜 필요한지
수강생이 진단 결과를 보고 우선순위를 판단하는데, 색상이 잘못되면 잘못된 의사결정을 하게 됨. 현재 거의 벤치마크에 근접한데도 빨간색(위험)으로 나와서 불필요한 경각심을 줌.

### 재현 조건
- 총가치각도기 → 진단 탭 → 전환율 섹션
- 구매전환율: 7.31% / 벤치마크: 7.72%
- 7.31 / 7.72 = 94.7% (벤치마크의 75% = 5.79 이상)
- 기대: 노란색 🟡 / 실제: 빨간색 🔴

### 기대 판정 기준
- 🟢 초록 (GOOD): 벤치마크 100% 이상
- 🟡 노란 (NORMAL): 벤치마크 75% 이상 ~ 100% 미만
- 🔴 빨간 (POOR): 벤치마크 75% 미만

### 파일
- `src/lib/diagnosis/engine.ts` (진단 판정 로직)
- `src/lib/protractor/` (프론트 표시 로직)

### 검증 기준
- 구매전환율 7.31% / 벤치마크 7.72% → 노란색 표시
- 벤치마크 75% 미만인 경우만 빨간색
- 다른 지표들도 동일 기준 적용 확인
- `npm run build` 성공
