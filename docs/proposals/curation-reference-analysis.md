# 큐레이션 탭 레퍼런스 분석

## 1. 주요 레퍼런스 서비스 분석

### Feedly (Pro+, $18/mo)
**핵심 패턴:**
- **AI Feed**: ML 모델이 토픽/키워드 조합으로 콘텐츠 자동 수집
- **Board**: Pinterest 스타일 — 토픽별 보드에 아티클 저장/분류
- **Clustering**: 같은 주제 기사 자동 그룹핑 ("Also in" 섹션)
- **Leo AI**: 중요도 우선순위 자동 정렬 + "Like-Board" 학습 (보드에 저장할수록 AI가 취향 학습)
- **뷰**: 매거진뷰 / 카드뷰 / 리스트뷰 전환

**우리가 배울 점:**
- 토픽 클러스터링은 자동이어야 함 (수동 태깅 X)
- Board = 토픽맵의 원형. 하지만 Feedly는 사용자가 직접 분류 → 우리는 AI 자동분류
- "Also in" 패턴 — 같은 주제 소스 묶어서 보여주기

---

### ContentStudio ($25/mo)
**핵심 패턴:**
- **Discover 탭**: 토픽 기반 콘텐츠 발견 → 한 줄 요약 + 소셜 반응 수치
- **Composer**: 발견한 콘텐츠를 바로 편집 → 발행
- **Planner**: 칸반 보드 (Idea → Draft → Review → Published)
- **Automation**: 토픽 → 자동 큐 → 승인 후 발행

**우리가 배울 점:**
- Discover → Compose 연결이 매끄러움 (클릭 한 번에 편집 모드)
- 칸반 보드 뷰가 상태 파이프라인 시각화에 효과적
- 소셜 반응 수치 = 우리의 importance_score

---

### Curata (Enterprise)
**핵심 패턴:**
- **Editorial Workflow**: 승인 플로우 + 브랜드 필터
- **AI Discovery**: ML이 관련 콘텐츠 추천 + 자동 태깅
- **Content Marketing Platform (CMP)**: 수집 → 큐레이션 → 발행 → 성과 추적 전체 루프
- **Source Attribution**: 원본 출처 + 생성물 양방향 추적

**우리가 배울 점:**
- 엔터프라이즈급이지만 핵심은 "파이프라인 전체 시각화"
- 수집 → 큐레이션 → 생성 → 발행 → 성과 각 단계가 한 화면에

---

### UpContent
**핵심 패턴:**
- **Topic + Query 조합**: 넓은 토픽 + 세부 키워드로 정밀 필터
- **Team Collaboration**: 댓글 + 공유 + 승인 워크플로
- **Automation Rules**: "hands-free" 큐레이션 (조건 충족 → 자동 승인)

**우리가 배울 점:**
- 토픽이 고정이 아니라 동적으로 바뀔 수 있어야 함
- 자동화 규칙 = 우리의 "importance_score ≥ 4이면 자동 생성" 같은 패턴

---

### Notion Editorial Calendar
**핵심 패턴:**
- **다중 뷰**: 칸반(상태별) / 캘린더(날짜별) / 테이블(빠른 편집) / 갤러리(비주얼) / 타임라인(캠페인)
- **Status Pipeline**: Idea → Draft → Edit → Approved → Published
- **Backlog + Sprint**: 아이디어 풀 + 이번 주기 작업 분리

**우리가 배울 점:**
- "같은 데이터, 다른 뷰" 패턴이 핵심
- 우리도 최소 3개 뷰: 리스트(기본) / 토픽맵(분류) / 칸반(상태)

## 3. Smith님 핵심 요구사항 반영

### "크롤링 하더라도 핵심요약을 볼 수 있어야"

**현재 문제**: 크롤링된 소스가 `body_md`에 원문이 있지만, 요약이 없거나 부실한 경우 많음

**해결 방안:**

```
[소스 카드 — 개선안]
┌────────────────────────────────────────────────┐
│ 🌐 블로그 크롤링                          ★★★★☆ │
│                                                │
│ "2026 메타 광고 ASC 강제 전환 — 자사몰 영향 분석" │
│                                                │
│ 📋 핵심요약 (AI 생성)                           │
│ ┌──────────────────────────────────────────┐   │
│ │ • ASC가 2026 Q2부터 모든 캠페인 기본값으로  │   │
│ │ • 수동 타겟팅 옵션 축소 → 소재 품질이 핵심  │   │
│ │ • 자사몰 대응: 크리에이티브 다변화 + CAPI    │   │
│ └──────────────────────────────────────────┘   │
│                                                │
│ 🏷️ 메타광고 · ASC · 타겟팅 · 크리에이티브       │
│ 📅 3/6 수집 | 출처: searchengineland.com         │
│                                                │
│ [원문 보기] [정보공유 생성] [스킵]               │
└────────────────────────────────────────────────┘
```

**구현:**
1. 크롤링 시점에 AI 요약 생성 (`ai_summary` 필드)
2. 기존 `ai_summary`가 null인 건 → 백필 스크립트로 채움
3. 카드에서 요약이 항상 보이게 (접히지 않은 상태)
4. 요약 = 불릿 3개 이내, 핵심만

### "분류마다 마인드맵처럼 보기"

**접근법 비교:**

| 방식 | 장점 | 단점 |
|------|------|------|
| A. 트리 (접을 수 있는 아코디언) | 구현 쉬움, 익숙한 UX | 시각적 임팩트 낮음 |
| B. 칸반 (토픽별 컬럼) | 상태 흐름 직관적 | 토픽 많으면 가로 스크롤 |
| C. 태그 클라우드 + 필터 | 빠른 탐색 | 구조 안 보임 |
| D. 트리맵 (영역 크기 = 콘텐츠 수) | 비주얼 임팩트 큼 | 클릭 후 디테일 봐야 함 |

**제안: A + B 하이브리드**
- 기본: **트리 뷰** (토픽 > 서브토픽 > 콘텐츠, 접기/펼치기) — 1차 목업에 반영한 것
- 추가: **칸반 뷰** (컬럼 = 상태, 카드 = 콘텐츠 + 토픽 태그)
- 향후: 트리맵이나 버블 차트는 콘텐츠 100개+ 넘어가면 고려

## 4. 현재 시스템과의 갭 분석

| 항목 | 현재 | 목표 |
|------|------|------|
| AI 요약 | `ai_summary` 있지만 일부 null | 100% 채움, 카드에서 항상 노출 |
| 토픽 분류 | `key_topics` 배열 (태그 수준) | 2-depth 토픽 트리 (카테고리 > 서브토픽) |
| 뷰 모드 | 리스트 1개 | 리스트 + 토픽맵 + 칸반 (3개) |
| 상태 | new/selected/dismissed/published | new/reviewing/generated/published/dismissed/archived (6개) |
| 삭제 | hard delete | soft delete + 소스 자동 복구 |
| 연결 | source_ref (단방향) | 양방향 (소스→생성물, 생성물→소스) |
| 클러스터링 | 없음 | AI 임베딩 기반 유사도 그룹핑 |

## 5. 다음 단계 제안

### Phase 0: 핵심요약 100% (1~2일)
1. `ai_summary` null인 기존 콘텐츠 백필
2. 크롤링 파이프라인에 요약 생성 스텝 추가
3. 카드 UI에서 요약 항상 노출

### Phase 1: 토픽 분류 + 뷰 (3~4일)
1. AI 토픽 자동 분류 (2-depth)
2. 트리 뷰(토픽맵) 구현
3. 상태 필터 탭

### Phase 2: 상태 머신 + 소프트 삭제 (2~3일)
1. 6단계 상태 전이
2. soft delete + 소스 복구
3. 양방향 연결 표시

### Phase 3: 칸반 뷰 + 고도화 (여유 시)
1. 칸반 뷰 (드래그 앤 드롭 상태 전이)
2. 벌크 액션
3. 클러스터링
