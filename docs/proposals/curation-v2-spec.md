## 1. 현재 상태

### 기존 큐레이션 탭
- **리스트 뷰 단일**: 날짜 그룹 (오늘/어제/이번주) + 카드
- **사이드바**: 소스별 필터 (blueprint/lecture/youtube/crawl/...)
- **카드**: 제목 + AI 요약(접혀있음, 클릭 시 펼침) + 중요도 ★ + 토픽 태그
- **액션**: 체크박스 선택 → 일괄 스킵 / 정보공유 생성

### 문제점
1. **AI 요약이 접혀있어서** 안 읽게 됨 → 요약 가치 소멸
2. **소스 성격 무시**: 블루프린트(커리큘럼) ≠ 블로그(시사) 근데 같은 뷰
3. **토픽 분류 없음**: 어떤 주제가 풍부하고 빈약한지 파악 불가
4. **칸반 없이도** 정보공유 탭에서 발행 관리 가능 (칸반 불필요)

### DB 현황
| 항목 | 수량 |
|------|------|
| 전체 콘텐츠 (contents) | 226 |
| ai_summary null | 30 (13%) |
| importance_score = 0 | 98 (43%) |
| source_type 분포 | info_share 14, case_study 12, webinar 2, lecture 1, ai 1 (null인 것 중) |

## 2. 목표

큐레이션 탭 = **"들어온 소스를 이해하고, 뭘 만들지 판단하는 곳"**
- 큐레이션 = **입구** (소스 이해 + 선별)
- 정보공유 = **출구** (생성 + 편집 + 발행)

## 3. 핵심 설계: 듀얼 모드

사이드바 소스 선택에 따라 메인 영역이 자동 전환:

```
[사이드바]                    [메인 영역]

📚 커리큘럼 소스
  🛡️ 블루프린트 ──────→  커리큘럼 뷰 (시퀀스 순서)
  🎓 자사몰사관학교 ───→  커리큘럼 뷰 (회차 순서)

🔍 큐레이션 소스
  🗄️ 전체 ──────────→  큐레이션 뷰 (인박스 or 토픽맵)
  🌐 블로그 ─────────→  큐레이션 뷰
  ▶️ YouTube ────────→  큐레이션 뷰
  📖 마케팅원론 ──────→  큐레이션 뷰
```

### 3.1 커리큘럼 모드 (블루프린트/자사몰사관학교)

**목적**: 체계적 교육 콘텐츠를 시퀀스로 관리하고, 순서대로 정보공유로 풀어내기

**특징**:
- 레벨/회차별 시퀀스 뷰 (1→2→3→...)
- 각 아이템 상태: ✓ 발행됨 / → 다음 발행 / 🔒 이전 완료 후
- 중복 콘텐츠 표시 ("초급 #2와 중급 #3, 30% 중복")
- 진행률 바 (2/3 발행)
- 시험 내용 제외, 강의 흐름과 핵심 내용만 표시

**블루프린트 구조**: 초급 → 중급 → 고급 (레벨 잠금)
**자사몰사관학교 구조**: 1회 → 2회 → 3회... (회차 순서)

**UI 컴포넌트**: `CurriculumView`
```
┌─────────────────────────────────────┐
│ 📚 커리큘럼 모드                       │
│                                     │
│ ┌─ 초급 · Meta 광고 기초 ─ 2/3 ████░ ─┐ │
│ │ ① 계정 구조와 캠페인 설정  ✓ 발행됨    │ │
│ │ ② 타겟팅 기본             ✓ 발행됨    │ │
│ │ ③ 픽셀과 전환 추적 📍     → 다음 발행  │ │
│ └────────────────────────────────┘ │
│                                     │
│ ┌─ 중급 · 성과 최적화 ─ 0/4 ░░░░░ ──┐ │
│ │ ① CBO 전략              🔒        │ │
│ │ ② 고급 타겟팅   ⚠️초급#2 30%중복  │ │
│ │ ③ A/B 테스트             🔒        │ │
│ │ ④ 어트리뷰션             🔒        │ │
│ └────────────────────────────────┘ │
└─────────────────────────────────────┘
```

### 3.2 큐레이션 모드 (외부 소스)

**목적**: 크롤링/수집된 외부 콘텐츠를 AI 기반으로 선별

**2가지 서브뷰**: 인박스 / 토픽맵 (토글 버튼)

#### 인박스 뷰
- 기존 리스트 뷰와 유사하되 **AI 요약 항상 펼침**
- 날짜 그룹: 오늘/어제/이번 주/그 이전
- 중요도순 정렬 (★5부터)

#### 토픽맵 뷰
- AI가 자동 분류한 토픽 트리: 대주제 > 소주제 > 콘텐츠
- 어떤 주제가 풍부/빈약한지 한눈에 파악
- 접기/펼치기

**UI 컴포넌트**: `CurationView` + `TopicMapView`
```
┌─ 📢 메타 광고 (14) ─────────────────┐
│  ├─ ASC / 어드밴티지+ (3)              │
│  │  ┌────────────────────────────┐  │
│  │  │ 🌐 브로드 vs ASC 비교 — 뷰티  ★★★★  │
│  │  │ 🤖 AI 핵심요약                      │
│  │  │  · 뷰티 3개 A/B 30일               │
│  │  │  · ASC: CPA↓ / 브로드: ROAS↑       │
│  │  │  · 단가 30만↑ 브로드 우세           │
│  │  │ [원문] [스킵] [생성]                │
│  │  └────────────────────────────┘  │
│  ├─ 타겟팅 전략 (4)                    │
│  └─ 소재 / 크리에이티브 (4)             │
└──────────────────────────────────────┘
```

## 4. 구현 페이즈

### Phase 0: 데이터 정비 (1~2일)
- `ai_summary` null 30건 백필 (AI 분석 재실행)
- `importance_score` 0인 98건 백필
- `key_topics` 빈 배열인 건 재분석
- 블루프린트/강의 → importance_score 5로 일괄 업데이트

### Phase 1: 사이드바 + 듀얼모드 전환 (2~3일)
- `PipelineSidebar` 개편: 커리큘럼 소스 / 큐레이션 소스 섹션 분리
- 소스 클릭 시 `CurriculumView` vs `CurationView` 조건부 렌더링
- `CurriculumView` 컴포넌트 신규 (시퀀스 뷰)
- DB: `contents` 테이블에 `sequence_order` 컬럼 추가 (nullable int)

### Phase 2: 큐레이션 뷰 리뉴얼 (2~3일)
- 인박스 + 토픽맵 서브뷰 토글
- `CurationCard` v2: AI 요약 항상 펼침 + 생성물 연결
- `TopicMapView` 컴포넌트 신규 (대주제>소주제 트리)
- 토픽 자동 분류: `key_topics` 기반 그룹핑 (클라이언트사이드)

### Phase 3: Soft Delete + 벌크 (1~2일)
- `curation_status` enum에 `'deleted'` 추가
- `deleted_at` 컬럼 추가
- 삭제 섹션 UI
- 복원 액션
- 30일 자동 정리 cron
