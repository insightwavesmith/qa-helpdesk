## 1. 목표

`collect-daily/route.ts`를 `collect-benchmarks/route.ts`와 동일한 GCP 기반 방식으로 정합성 맞추기.
현재 불일치 항목:
- `video_p3s_rate` 분모: reach → impressions
- `retention_rate`: thruplay/videoP3s → video_p100/videoP3s
- 랭킹 3종 미수집: quality_ranking, engagement_ranking, conversion_ranking
- `video_p100` 미수집/미저장
- ACTIVE 광고 필터 없음

## 6. normalizeRanking 함수

collect-benchmarks에서 동일 함수 복사:

```typescript
function normalizeRanking(raw: string | undefined | null): string {
  if (!raw) return 'UNKNOWN';
  const u = raw.toUpperCase().replace(/-/g, '_');
  if (u.includes('ABOVE')) return 'ABOVE_AVERAGE';
  if (u.includes('BELOW')) return 'BELOW_AVERAGE';
  if (u === 'AVERAGE') return 'AVERAGE';
  return 'UNKNOWN';
}
```

## TDD 보완 (테스트 주도 개발 지원)

### T1. 단위 테스트 시나리오

| 함수 | 입력 | 기대 출력 | 검증 포인트 |
|------|------|----------|------------|
| `normalizeRanking("ABOVE_AVERAGE")` | `"ABOVE_AVERAGE"` | `"ABOVE_AVERAGE"` | 정상 값 그대로 반환 |
| `normalizeRanking("above-average")` | `"above-average"` (하이픈) | `"ABOVE_AVERAGE"` | 하이픈→언더스코어 변환 + 대소문자 처리 |
| `normalizeRanking("AVERAGE")` | `"AVERAGE"` | `"AVERAGE"` | 정확 일치 |
| `normalizeRanking("below_average")` | `"below_average"` (소문자) | `"BELOW_AVERAGE"` | 소문자 처리 |
| `normalizeRanking(null)` | `null` | `"UNKNOWN"` | null 안전 처리 |
| `normalizeRanking(undefined)` | `undefined` | `"UNKNOWN"` | undefined 안전 처리 |
| `normalizeRanking("")` | `""` | `"UNKNOWN"` | 빈 문자열 처리 |
| `calculateMetrics()` — video_p3s_rate | `{ videoP3s: 500, impressions: 10000 }` | `video_p3s_rate = 5.0` | 분모가 impressions (reach 아님) |
| `calculateMetrics()` — retention_rate | `{ video_p100: 200, videoP3s: 1000 }` | `retention_rate = 20.0` | 분모가 videoP3s, 분자가 video_p100 (thruplay 아님) |
| `calculateMetrics()` — 0 나누기 방지 | `{ videoP3s: 0, impressions: 0 }` | `video_p3s_rate = 0, retention_rate = 0` | NULLIF 또는 0 체크 |
