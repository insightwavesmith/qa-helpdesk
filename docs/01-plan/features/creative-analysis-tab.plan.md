# 소재 분석 탭 — Plan (목업 설계 제안서)

## 타입: 목업
> src/ 코드 파일 수정 금지. HTML 목업 + 설계 문서만 작성.

## 이게 뭔지
총가치각도기(Protractor)에 **'소재 분석'** 탭을 추가하여, 소재 건강도·일관성·클러스터·고성과 패턴을 시각적으로 보여주는 UI.

## 왜 필요한지
- Phase 1~2에서 소재 임베딩(3072차원), LP 크롤링, 일관성 점수를 모두 구축했지만 **사용자가 볼 수 있는 UI가 없음**
- 현재 데이터: 835건 소재, 337건 임베딩, 90건 LP 구조 분석, 80건 일관성 점수
- 수강생(student)과 관리자(admin)가 자기 광고계정의 소재 상태를 한눈에 파악하고 개선 방향을 잡을 수 있어야 함
- Phase 3 (영상 VLM 해석: Gemini 2.0 Pro) 전에 UI 프레임을 확정해야 데이터 파이프라인 설계가 구체화됨

## 기존 UI 구조 분석

### 탭 네비게이션 (`protractor-tab-nav.tsx`)
```
[대시보드] [경쟁사 분석] ← 여기에 [소재 분석] 추가
```
- Primary color 하단 보더(`border-[#F75D5D]`) 활성 표시
- `<Link>` 기반 라우팅 (`/protractor/creatives`)

### 레이아웃 (`layout.tsx`)
- `max-w-6xl mx-auto px-4 py-8`
- 역할 기반 접근 제어 (lead 차단, member/student/admin 허용)
- `<ProtractorTabNav />` + `{children}`

### 대시보드 패턴 (`real-dashboard.tsx`)
- **ProtractorHeader**: 계정 선택 드롭다운 + 날짜 범위
- **PeriodTabs**: 1일/7일/14일/30일 기간 선택
- **Tabs (shadcn/ui)**: 성과 요약 / 콘텐츠 서브탭
- **SWR 데이터 페칭** + Skeleton 로딩
- 카드 스타일: `rounded-2xl border border-gray-200 bg-white p-4`

### 디자인 시스템
- Primary: `#F75D5D`, hover: `#E54949`
- 폰트: Pretendard
- 배경: 흰색, 라이트 모드만
- 카드: `rounded-2xl border border-gray-200 bg-white`
- 참고: Triple Whale 스타일

┌──────────────────────┐ ┌──────────────────────────┐

- `FatigueTopList` — 테이블 (썸네일, 제목, 유형, 피로도 점수, ROAS)

└──────────────────────┘ └──────────────────────────┘

- `ConsistencyOverview` — 도넛/반원 게이지 (계정 평균)

- `ProblemCreativeList` — 일관성 낮은 소재 테이블 (threshold: 0.4)
