## Executive Summary

CTO 세션이 CLAUDE.md 규칙을 무시하고 PM Plan 없이 구현 시도 + PM Do 지시에도 COO에 재확인.
원인: CLAUDE.md 규칙이 분산되어 CTO가 새 세션에서 핵심 규칙을 놓침.
해결: CTO/PM 역할별 필수 행동을 명시적으로 집약 + 위반 사례 안티패턴 추가.

## 해결 방안

### 변경 1: CTO 필수 행동 섹션 신설

CLAUDE.md에 **"CTO 필수 행동 (위반 시 리젝)"** 섹션 추가.
- 흩어진 CTO 관련 규칙을 하나로 집약
- DO/DON'T 구조로 명확화
- 오늘 발생한 위반 사례를 안티패턴으로 기록

### 변경 2: PM→CTO 핸드오프 규칙 명시

"PM이 Do 지시하면" 규칙을 CTO 섹션에도 중복 배치.
자동진행 섹션에만 있으면 CTO가 놓침.

### 변경 3: 세션 시작 필수 읽기 강화

CTO 세션이 세션 시작 시 반드시 확인할 체크포인트 명시.
