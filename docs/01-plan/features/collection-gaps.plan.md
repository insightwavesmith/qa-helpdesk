## 1. 배경 및 목적

### 배경
Phase 1 코드 구현은 거의 완료(Match Rate 98%)되었으나, 실제 데이터 수집/분석 파이프라인에 상당한 갭이 존재한다. 수집-임베딩-분석의 3단계 파이프라인 중 수집과 임베딩은 95%+ 완료되었으나, 핵심 분석 단계인 5축 분석이 16%에 머물러 있어 서비스의 핵심 가치인 "왜 부족한지 + 어떻게 고칠지"를 제공할 수 없는 상태이다.

### 현재 숫자 (2026-03-25 기준, DEV-STATUS.md 참조)

| 항목 | 완료 | 전체 | 비율 | 갭 |
|------|-----:|-----:|-----:|-----|
| 등록 계정 | 23 | 42 | 55% | 19개 미수집 |
| 소재 수집 | 3,022 | 3,022+ | 100% | — |
| 임베딩 (768차원) | 2,917 | 3,022 | 97% | 105건 미임베딩 |
| DeepGaze 시선 히트맵 | 2,926 | 3,022 | 97% | 96건 미분석 |
| 미디어 Storage | 2,873+ | 3,022 | 95%+ | 83건+ storage 없음 |
| **Gemini 5축 분석** | **496** | **3,022** | **16%** | **2,526건 미분석 (핵심 갭)** |
| LP 크롤링 | 1,796+ | 1,796+ | 100% | — |
| LP 분석 (8카테고리) | 84 | 216 | 39% | 132건 미분석 |

### 목적
1. 미수집 19개 계정의 원인을 파악하고 해결 가능한 것은 수집 재개
2. 5축 분석 2,526건을 배치 실행하여 16% → 100% 달성
3. LP 미분석 132건을 배치 실행하여 100% 달성
4. LP 미디어 리소스(GIF/이미지/영상) 다운로드로 LP 보존 완성
5. 나머지 갭(storage 83건, 미임베딩 105건) 해소

## 5. 의존성

| 서브태스크 | 선행 의존 | 비고 |
|-----------|----------|------|
| TASK 1 (미수집 원인) | 없음 | 독립 실행 가능 |
| TASK 2 (5축 배치) | TASK 5-storage (미디어 있어야 분석 가능) | storage 없는 83건 먼저 해결 |
| TASK 3 (LP 배치) | 없음 | LP 크롤링 이미 100% 완료 |
| TASK 4 (LP 미디어) | 없음 | 독립 실행 가능, 단 DB migration 필요 |
| TASK 5 (나머지 갭) | 없음 | 독립 실행 가능 |

### 외부 의존성
- **Gemini API**: 5축 분석 + LP 분석에 필수. 과금 발생 (~$10 예상)
- **Meta API**: 미수집 계정 원인 파악 + 미디어 다운로드에 필요
- **GCS (bscamp-creatives)**: 미디어 저장소
- **Cloud Run Jobs**: 대량 배치 실행 환경
- **Railway DeepGaze**: LP 시선 분석 (TASK 3 부분)
