# 인라인 편집기 계획서

## 1. 개요
- 관리자가 실제 게시글 화면에서 바로 편집 가능
- 별도 admin 패널이 아니라 보이는 그대로(WYSIWYG) 편집
- TipTap 기반 인라인 에디터 → MDXEditor로 교체 결정 (2026-02-10)

## 2. 범위

### 포함
- /posts/[id] 페이지: admin일 때 "수정" 버튼 노출
- 인라인 에디터 활성화 (본문, 제목, 카테고리, 태그)
- 이미지 교체/추가
- "발행" / "이메일 발송" 버튼
- 상태 관리 (draft → published)
- 뉴스레터 미리보기 편집

### 제외
- 새 글 작성 (콘텐츠 관리에서)
- 이메일 발송 로직 자체 (content-pipeline에서 구현)

## 3. 성공 기준
- [ ] 관리자가 게시글 상세에서 바로 편집 가능
- [ ] 마크다운 포맷(테이블, 넘버링, 인용문) 유지
- [ ] 편집 후 저장 → 즉시 반영
- [ ] published 글 편집 시 상태 유지 (초안으로 변경 X)

## 4. 의존성
- Phase 1 완료 (페이지-데이터 연결)
- 콘텐츠 파이프라인 (contents 테이블 구조)

## 5. 이슈 & 결정
- **TipTap 포맷 손실 문제** (2026-02-10 발견): 마크다운→HTML→TipTap→저장 시 테이블/넘버링/인용문 전면 깨짐
- **결정**: MDXEditor로 교체 (마크다운 네이티브 WYSIWYG)
- **상태 버그**: published 글 편집 시 "초안"으로 표시됨 → 수정 필요

## 6. 완료 상태
- **Phase**: do (TipTap 버전 구현 완료, MDXEditor 교체 진행중)
- Phase 3a로 초기 버전 배포 완료 (2026-02-09~10)
