## 1. 개요

### 기능 설명
Phase 1 코드 구현은 거의 완료됐으나, 실제 데이터 수집과 분석에 큰 갭이 존재한다.
코드 문제가 아니라 **배치 실행 + 원인 파악**이 핵심이다.

수집~임베딩~분석 파이프라인의 각 단계에서 미처리 데이터를 식별하고,
원인을 분류한 뒤 배치 실행으로 갭을 해소하는 것이 이 계획의 목적이다.

### 해결하려는 문제
1. **계정 수집 갭**: 42개 등록 계정 중 19개가 미수집 상태. 원인 불명.
2. **소재 분석 갭**: 3,022건 중 2,526건(84%)이 Gemini 5축 미분석.
3. **LP 분석 갭**: 216건 중 132건(61%)이 미분석.
4. **Storage 결손**: 소재 83건이 Storage에 미디어 파일 없음.
5. **임베딩 결손**: 소재 105건이 미임베딩.
6. **특정 계정 수집 중단**: 글로우빈/프로이덴이 3/19 이후 수집 멈춤.

## 3. TASK 목록 (5개)

### TASK 1: 미수집 19개 계정 원인 파악 (P0 - 최우선)

**이게 뭔지**: 42개 등록 계정 중 23개만 성과 수집되는 원인을 파악하고 분류하는 것

**왜 필요한지**: 19개 계정의 소재/성과 데이터가 수집되지 않으면 해당 수강생은 총가치각도기를 쓸 수 없다. 코드 버그인지, 권한 문제인지, 정상인지 분류해야 후속 조치가 가능하다.

**구현 내용**:
1. `ad_accounts` 테이블에서 `active=true`인데 `daily_ad_insights`에 데이터 없는 계정 목록 추출
2. 각 계정의 `meta_ad_account_id`로 Meta API 접근 가능 여부 확인 (토큰 권한)
3. 원인 분류:
   - **(a) Meta 권한 없음**: 목록 정리 → Smith님께 보고 (수동 권한 추가 필요)
   - **(b) 광고 안 돌리는 중**: 정상 — 수집할 데이터 없음. 상태만 기록.
   - **(c) collect-daily 코드 버그**: 코드 수정 필요
4. 글로우빈/프로이덴 3/19 수집 중단 원인 별도 확인 (토큰 만료? 계정 비활성화?)

**산출물**: 계정별 원인 분류표 (CSV 또는 보고서)

**예상 소요**: 2~3시간

### TASK 2: 미분석 소재 2,526건 Gemini 5축 배치 (P1)

**이게 뭔지**: 5축 분석(시각, 카피, 구조, 감정, 오디오)이 안 된 소재 2,526건을 Gemini로 일괄 분석하는 것

**왜 필요한지**: 현재 5축 분석률이 16%(496/3,022)로, 총가치각도기의 핵심 데이터가 대부분 비어있다. 90% 이상 도달해야 수강생에게 유의미한 진단 결과를 보여줄 수 있다.

**구현 내용**:
1. `analyze-five-axis.mjs`로 미분석 소재 배치 실행
2. GCP Cloud Run Job `bscamp-analyze-five-axis` 트리거 또는 로컬 배치 실행
3. 이미지와 영상 분리 처리:
   - 이미지: 건당 ~$0.003, 빠름
   - 영상: 건당 ~$0.015, 느림 (VLM 분석)
4. 진행률 로그 남기기 (100건 단위)
5. 실패 건 재시도 로직 확인 (Gemini API 429 대응)

**비용 추정**:
- 이미지 2,870건 중 미분석분 × $0.003 = ~$7.2
- 영상 152건 중 미분석분 × $0.015 = ~$2.3
- 합계: **$7.6 ~ $37.9** (전체 미분석분 기준 최대)

**산출물**: 5축 분석률 16% → 90%+

**예상 소요**: 4~6시간 (API 호출 대기 시간 포함)

### TASK 3: LP 미분석 132건 배치 분석 (P1)

**이게 뭔지**: LP(랜딩페이지) 216건 중 미분석 132건을 Gemini 8카테고리 분석으로 처리하는 것

**왜 필요한지**: LP 분석률이 39%로, 소재-LP 일관성 분석(creative-lp-alignment)과 LP 교차분석(lp-data-analysis) 기능이 제대로 동작하지 않는다.

**구현 내용**:
1. GCP Cloud Run Job `job-analyze-lps` 트리거
2. 또는 로컬에서 `analyze-lps-v2.mjs` 배치 실행
3. LP saliency(DeepGaze) 미분석분도 함께 처리
4. HTML 기반 분석이므로 `lp_snapshots`에 HTML이 있는지 사전 확인
   - HTML 없는 LP는 `crawl-lps` 재크롤링 선행 필요

**산출물**: LP 분석률 39% → 90%+

**예상 소요**: 2~3시간

### TASK 4: LP 미디어 다운로드 (P2)

**이게 뭔지**: LP HTML 안에 참조된 이미지/GIF/영상을 실제로 다운로드해서 GCS에 저장하는 것

**왜 필요한지**: 현재 LP는 스크린샷+HTML만 보존. GIF 애니메이션, 상품 영상 등 핵심 미디어가 빠져있어 LP가 변경/삭제되면 원본을 볼 수 없다. 시계열 비교와 정밀 분석에 필요.

**구현 내용** (상세: `.claude/tasks/TASK-LP-MEDIA-DOWNLOAD.md` 참조):
1. LP HTML 파싱 → `<img>`, `<video>`, `<source>`, CSS `background-image` 등에서 미디어 URL 추출
2. 미디어 파일을 GCS에 다운로드: `lp/{account_id}/{lp_id}/media/{hash}.{ext}`
3. DB `landing_pages` 테이블에 `media_assets` JSONB 컬럼 추가 (원본URL → Storage경로 매핑)
4. hash 비교로 중복 다운로드 방지
5. 파일 크기 제한: 개별 50MB, LP당 총 200MB
6. 대상: jpg, png, webp, svg, gif, mp4, webm (1x1 픽셀, favicon, 트래커 제외)

**주의사항**:
- 카페24 상세페이지 특성: 이미지 대부분 `ecimg.cafe24img.com` CDN
- 크론 실행 시간이 길어지면 별도 배치로 분리 검토
- 계정 종속 Storage 경로 준수 (ADR-001)

**산출물**: LP 미디어 보존 체계 구축

**예상 소요**: 4~6시간 (코드 구현 + 배치 실행)

### TASK 5: 나머지 갭 해소 (P2)

**이게 뭔지**: Storage 결손 83건, 미임베딩 105건, 특정 계정 수집 중단 등 잔여 갭을 모두 해소하는 것

**왜 필요한지**: 97% 완료율이지만 남은 3%가 특정 계정/소재에 집중돼 있을 수 있다. 완전한 데이터셋이 있어야 분석 결과의 신뢰도가 보장된다.

**구현 내용**:
1. Storage 없는 소재 83건 → `download-missing-media.mjs` 실행
2. 미임베딩 105건 → `embed-creatives` 배치 실행 (Cloud Run Job `bscamp-embed-creatives` 또는 로컬)
3. 글로우빈/프로이덴 수집 멈춤 원인 확인 (TASK 1 결과에 따라 조치)
4. DeepGaze 미분석 96건 처리 (Railway DeepGaze 배치)

**산출물**:
- Storage 결손: 83건 → 0건
- 미임베딩: 105건 → 0건
- DeepGaze 미분석: 96건 → 0건

**예상 소요**: 2~3시간

## 10. Executive Summary

Phase 1 코드는 완성됐으나 실제 파이프라인 실행에 갭이 크다.
**5축 분석 16%, LP 분석 39%, 미수집 계정 45%** — 이 세 가지가 핵심 병목이다.

이 계획은 코드 신규 구현이 아닌 **배치 실행 + 원인 진단**에 초점을 맞춘다.
총 비용 $10~$38, 예상 소요 2~3일이면 전체 분석률을 90%+ 수준으로 끌어올릴 수 있다.

TASK 1(계정 원인 파악)을 최우선으로 실행하고,
이후 TASK 2(5축)와 TASK 5(나머지 갭)를 병렬로 진행하는 것을 권장한다.
