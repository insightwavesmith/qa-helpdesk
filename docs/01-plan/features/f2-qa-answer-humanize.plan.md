# F2. AI 답변 프롬프트 humanize — Plan

> 작성: 2026-03-04
> 선행 작업: B3(ai-prompt-humanize) — 정보공유 프롬프트에 이미 적용 완료
> 참조: TASK.md F2

## 1. 개요
- **기능**: 수강생 질문에 대한 AI 자동 답변 톤을 자연스럽게 변경
- **해결하려는 문제**: AI 답변이 반복적인 패턴을 보임
  - "안녕하세요!" 인사말 시작
  - "~드리겠습니다" / "설명드리겠습니다" 반복
  - "더 궁금한 점이 있으시면 언제든 추가 질문해주세요! 😊" 마무리
  - 과도한 마크다운 (`##`, `**`, ✅ 이모지 남발)
- **수정 대상**: `src/lib/knowledge.ts` — `QA_SYSTEM_PROMPT`

## 2. 현재 상태 분석

### QA_SYSTEM_PROMPT 현재 상태 (knowledge.ts L83-148)
현재 프롬프트에는 이미 B3 태스크에서 humanize 규칙이 **대폭 추가되어 있음**:
- 말투 규칙 (단정형 위주, ~요 30% 이하)
- AI 상투어 금지 목록
- 어미 다양화 규칙
- 문장 리듬
- 경험담 톤
- 톤 레퍼런스 예시
- 셀프 검수 체크리스트

### TASK.md 요구사항 vs 현재 프롬프트 대조

| TASK.md 요구 | 현재 프롬프트 | 상태 |
|-------------|-------------|------|
| 인사말/맺음말 제거 — 바로 본론 | ✅ L90: "AI식 인사 금지. 바로 본론부터" | 이미 적용 |
| ~요체 30% 이하, 단정형 위주 | ✅ L87: "~요 어미는 전체 30% 이하" | 이미 적용 |

| 마크다운 최소화 (굵기만, ## 금지) | ⚠️ 현재 ## 금지 규칙 **없음** | **추가 필요** |
| 톤: 간결하고 실용적 | ✅ L92: "핵심만 짧게" | 이미 적용 |

### Gap: 마크다운 사용 규칙
현재 프롬프트에 마크다운 포맷팅 제한 규칙이 없음. AI가 여전히 `##` 헤딩, 과도한 불릿, 체크마크(✅) 등을 사용할 가능성 있음.
