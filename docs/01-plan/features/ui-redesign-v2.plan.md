# UI 리디자인 V2 계획서

## 1. 개요
- Q&A 헬프데스크 UI를 교육 플랫폼에 맞는 따뜻하고 깔끔한 디자인으로 전면 리디자인
- V1(노션 스타일)은 너무 밋밋하고 교육 서비스 느낌이 안 남 → 거부됨

## 2. 핵심 요구사항

### 기능적 요구사항
- FR-01: 카드 기반 Q&A 리스트 (푸딩캠프 스타일 그리드)
- FR-02: 카테고리 태그 필터 (칩/탭 형태)
- FR-03: 검색바 (히어로 영역 or 상단 고정)
- FR-04: 질문 상태 배지 (답변 대기 / 답변 완료)
- FR-05: 반응형 (모바일 1열 → 태블릿 2열 → 데스크탑 3열)
- FR-06: 로그인/회원가입 페이지도 동일 톤앤매너

### 비기능적 요구사항
- NFR-01: 모바일 퍼스트 (수강생 대부분 모바일 접속 예상)
- NFR-02: 로딩 속도 유지 (현재 수준 이상)
- NFR-03: 접근성 (색상 대비, 폰트 크기)

## 4. 현재 UI 문제점 (V1 노션 스타일)
- 너무 플랫하고 차가움 → 교육 서비스 느낌 안 남
- 장식 요소를 너무 과하게 제거 → 밋밋함
- 사이드바 구조가 수강생에게는 과한 네비게이션
- Smith님 피드백: "이건 아닌 것 같아"

## 5. 범위
### 포함
- 로그인/회원가입 페이지
- 홈 (공지 + Q&A 미리보기)
- Q&A 리스트 (전체/내 질문)
- Q&A 상세 페이지
- 질문 작성 폼
- 관리자 대시보드
- 사이드바/네비게이션

### 제외
- 백엔드 로직 변경 없음
- DB 스키마 변경 없음
- 기능 추가 없음 (순수 UI 변경)

## 6. 성공 기준
- [ ] Smith님 목업 확인 후 승인
- [ ] 모바일에서 사용성 문제 없음
- [ ] 푸딩캠프/인프런 수준의 깔끔한 비주얼
- [ ] 기존 기능 100% 동작

## 7. 프로세스 (bkit PDCA 엄격 적용)
1. Plan (이 문서) ✅
2. Design → 디자인 시스템 정의 (컬러, 폰트, 간격, 컴포넌트)
3. 목업 → HTML/CSS 목업 or 스크린샷 → Smith님 확인
4. Do → 승인 후 구현
5. Check → Gap 분석
6. Act → 수정 후 배포

**중요: 목업 승인 전에 절대 코딩 시작하지 않음**
