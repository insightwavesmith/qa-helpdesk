# 꼬리질문 기능 계획서

> 작성: 2026-03-16

## 기능 설명
질문에 답변이 달린 후, 수강생이 해당 답변에 대해 추가 질문(꼬리질문)을 할 수 있는 기능.

## 왜 필요한지
- 수강생이 답변을 받고도 추가 궁금한 점이 있으면 새 질문을 작성해야 함
- 맥락이 끊기면 AI가 이전 대화를 참조하지 못해 답변 품질 저하
- 꼬리질문+답변 스레드를 하나의 맥락으로 임베딩하면 RAG 검색 품질 향상

## 구현 방식: B안 (parent_question_id)
- questions 테이블에 `parent_question_id` 컬럼 추가 (nullable FK)
- createQuestion에 `parentQuestionId` 옵션 추가 → 기존 AI 자동답변 파이프라인 재사용
- 질문 상세 페이지에서 스레드 형태로 꼬리질문+답변 표시

## 핵심 요구사항
1. **안전성**: `parent_question_id` 컬럼 없어도 기존 기능 100% 동작
2. **스레드 임베딩**: 꼬리질문 답변 승인 시 원본 스레드 전체를 하나의 맥락으로 임베딩
3. **getQuestions() 수정 금지**: 장애 원인이었던 `.is()` 필터 추가 안 함

## 성공 기준
- tsc/build 통과
- 기존 질문 목록/상세 페이지 정상 (migration 없이도)
- 꼬리질문 작성 시 AI 자동답변 동작
- 답변 승인 시 스레드 전체 임베딩 (`qa_thread` source_type)
