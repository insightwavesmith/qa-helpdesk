# T5. 정보공유 AI 프롬프트 개선 — Plan

## 1. 개요
- **기능**: education 타입 AI 프롬프트에 상단 3줄 요약, 핵심 숫자 블록, 체크리스트, 섹션 구분선 추가
- **해결하려는 문제**: AI 생성 콘텐츠의 구조가 단조롭고, T4 CSS 스타일과 연계할 마크다운 패턴이 부족
- **위치**: `src/actions/contents.ts` 533~548줄 (education 프롬프트)

## 2. 핵심 요구사항

### 기능적 요구사항
- FR-01: **상단 3줄 요약 박스** — 글 시작 전 `> **📌 핵심 요약**` blockquote 형태
- FR-02: **핵심 숫자 블록** — 섹션 시작 시 `- **6억 건** — 설명` 형태의 핵심 수치
- FR-03: **체크리스트** — 실무 점검 항목은 `- ✅ ~하고 있나요?` 형태
- FR-04: **섹션 구분** — `---` 구분선을 각 h2 앞에 필수 삽입

### 비기능적 요구사항
- 기존 구조(넘버링 h2, blockquote 하이라이트, 볼드 팁) 유지
- CONTENT_BASE_STYLE의 메타 광고 지식 섹션 변경 금지
- 다른 타입(case_study, webinar 등) 프롬프트 변경 금지
- RAG 검색 로직(knowledge.ts) 변경 금지

## 3. 범위

### 포함
- `contents.ts`의 `TYPE_PROMPTS.education.system` 프롬프트 텍스트 수정

### 제외
- CONTENT_BASE_STYLE 변경
- case_study, webinar 등 다른 타입 프롬프트
- knowledge.ts (RAG 검색 로직)
- 마크다운 변환 로직 (post-body.tsx)

## 4. 성공 기준
- [ ] 새로 생성한 education 콘텐츠에 상단 3줄 요약 blockquote가 포함된다
- [ ] 각 섹션 시작 시 핵심 수치가 볼드 리스트로 포함된다
- [ ] 실무 점검 항목이 ✅ 체크리스트 형태로 포함된다
- [ ] 각 h2 앞에 `---` 구분선이 포함된다
- [ ] 기존 구조(넘버링 h2, blockquote, 볼드 팁)가 유지된다
- [ ] 다른 타입 프롬프트에 영향이 없다
- [ ] `npm run build` 성공
