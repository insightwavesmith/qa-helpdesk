# 회원가입 폼 클라이언트 Validation 강화 (T1) Plan

## 요구사항
회원가입 폼(`signup/page.tsx`)의 클라이언트 사이드 유효성 검사를 강화한다.
현재는 비밀번호 불일치/8자 미만/사업자번호 빈값만 JS로 체크하고, 나머지는 HTML `required` + `type="email"`에 의존한다.
Supabase signUp 호출 전 프론트에서 모든 형식 오류를 잡아 UX를 개선한다.

## 범위
- **파일**: `src/app/(auth)/signup/page.tsx` (수정만)
- **의존**: 없음 (기존 signup-refactor 완료 상태에서 validation만 추가)
- **제외**: 회원가입 전체 로직/흐름 변경 금지, B1 수정(authError + user 존재 시 진행) 건드리지 않음, 이메일 중복은 T2에서 처리

## 현재 문제점
1. 빈 필드 → HTML `required`만으로는 에러 위치가 불명확
2. 이메일 형식 → `type="email"`만 의존, 커스텀 피드백 없음
3. 전화번호 형식 (lead 모드) → 검사 없음
4. 사업자번호 형식 (lead 모드) → 빈값만 체크, 형식 검사 없음
5. 비밀번호 확인 → submit 시에만 체크, 실시간 피드백 없음
6. submit 버튼 → lead 모드에서 사업자번호만 체크, 나머지 필수 필드 미검증
7. 에러 표시 → 폼 상단에 1개만 표시, 어떤 필드가 문제인지 불명확

## 기대 동작
1. **필수값 체크**: submit 시 빈 필드 아래에 "필수 항목입니다" 빨간 텍스트
2. **이메일 형식**: 정규식 검사 → "올바른 이메일 형식이 아닙니다"
3. **전화번호 형식** (lead): `01X-XXXX-XXXX` 또는 `01XXXXXXXXX` → "올바른 전화번호 형식이 아닙니다" (자동 하이픈 포맷팅 포함)
4. **사업자번호 형식** (lead): `XXX-XX-XXXXX` (10자리 숫자) → "올바른 사업자등록번호 형식이 아닙니다" (자동 하이픈 포맷팅 포함)
5. **비밀번호 확인**: 입력 중 실시간 불일치 표시
6. **submit 버튼**: 모든 필수 필드 미입력 시 disabled
7. **에러 표시**: 각 필드 아래 개별 에러 메시지, 상단 에러 박스는 서버 에러(Supabase 등)용으로만 유지

## 성공 기준
1. 빈 필드 제출 시 각 필드 하단에 개별 에러 메시지 노출
2. 이메일 형식 오류 → 필드 하단 에러 메시지
3. 전화번호 형식 오류 → 필드 하단 에러 메시지 + 자동 하이픈
4. 사업자번호 형식 오류 → 필드 하단 에러 메시지 + 자동 하이픈
5. 비밀번호 불일치 → 실시간 필드 하단 에러 메시지
6. 서버 에러(Supabase)만 상단 에러 박스에 표시
7. 기존 가입 플로우(student/lead 분기, 초대코드 등) 무변경
8. `npm run build` 성공
