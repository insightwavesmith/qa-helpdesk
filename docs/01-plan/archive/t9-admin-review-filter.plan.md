# T9. 관리자 후기 목록 필터 UI 누락 — Plan

## 1. 개요
- **기능**: 관리자 후기 목록(`/admin/reviews`)에 기수 필터 + 카테고리 필터 드롭다운 추가
- **해결하려는 문제**: 관리자 페이지에 필터 UI가 없어 전체 후기를 스크롤하며 찾아야 함. 수강생 페이지에는 이미 기수/카테고리 필터가 존재
- **원인**: `/admin/reviews/page.tsx`에서 `getReviewsAdmin()`으로 전체 데이터를 가져온 후 필터링 없이 테이블에 표시

## 2. 핵심 요구사항

### 기능적 요구사항
- FR-01: 관리자 후기 목록 상단에 **기수 필터** 드롭다운 추가 (전체 / 1기 / 2기 / 3기 / 4기 / 5기)
- FR-02: **카테고리 필터** 드롭다운 추가 (전체 / 일반후기 / 졸업후기 / 주차별 후기)
- FR-03: 필터 선택 시 목록이 **즉시 갱신**된다 (클라이언트 필터링)
- FR-04: 필터 결과가 0건이면 "해당 조건의 후기가 없습니다" 빈 상태 표시
- FR-05: 필터 적용 상태에서도 고정(pin) / 삭제 기능이 정상 작동한다

### 비기능적 요구사항
- 필터 UI 스타일은 수강생 페이지(`review-list-client.tsx`)의 select 스타일과 동일
- API 엔드포인트 추가 금지 — 기존 `getReviewsAdmin()` 데이터에서 클라이언트 필터링
- 수강생 후기 페이지 변경 금지

## 3. 범위

### 포함
- 관리자 후기 목록 상단에 필터 바 UI 추가 (기수 + 카테고리 select)
- 클라이언트 상태 기반 필터링 로직 (useState + useMemo/filter)
- 필터 적용된 건수 표시
- 빈 상태 UI (필터 결과 없을 때)

### 제외
- 서버 사이드 필터링 (기존 getReviewsAdmin API 변경 없음)
- 정렬 기능 (관리자 목록은 고정→최신순 고정)
- 검색 기능 (텍스트 검색)
- 수강생 후기 페이지 수정
- 새 API 엔드포인트 추가

## 4. 성공 기준
- [ ] 기수 드롭다운으로 특정 기수 후기만 필터링할 수 있다
- [ ] 카테고리 드롭다운으로 특정 카테고리 후기만 필터링할 수 있다
- [ ] 기수 + 카테고리 동시 필터링이 가능하다
- [ ] 필터 스타일이 수강생 페이지 필터와 동일하다
- [ ] 필터 결과 0건 시 빈 상태 메시지가 표시된다
- [ ] `npm run build` 성공
