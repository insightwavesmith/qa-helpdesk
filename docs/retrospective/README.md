## 교훈 목록

### RET-001: Big Bang 마이그레이션 금지 (2026-03-26)
**사고**: Supabase SDK → Cloud SQL 전환 시 126파일 동시 변경 (d86a923)
**결과**: BUG 9건, 25파일 INSERT 무효화, 사이트 쓰기 기능 전면 장애
**교훈**:
- 대규모 전환은 3단계 이상 분할 (Auth → DB → 패키지 제거)
- 각 단계마다 런타임 QA 필수 (tsc+build만으로 불충분)
- 체이닝 패턴 전수 조사 없이 SDK 교체 금지
**상세**: [2026-03-26-query-builder-postmortem.md](2026-03-26-query-builder-postmortem.md)

### RET-002: 런타임 QA 없이 merge 금지 (2026-03-26)
**사고**: tsc+build 통과 → merge → 런타임에서 SQL 생성 오류
**결과**: `.insert().select()`가 INSERT 대신 SELECT 실행. 에러 없이 데이터 유실.
**교훈**:
- SQL 문자열 생성 정확성은 타입 체크로 검증 불가
- 마이그레이션 PR에는 런타임 체크리스트 필수 (로그인→글작성→조회→댓글)
- "에러가 안 나는 버그"가 가장 위험함

### RET-003: 팀원 산출물 중간 검증 필수 (2026-03-26)
**사고**: backend-dev에게 위임 후 커밋까지 방치 → BUG-3 누락 발견 못 함
**결과**: 수정 완료라고 보고했으나 가장 중요한 버그가 빠져있었음
**교훈**:
- 팀원 커밋 전에 diff 확인 (최소 핵심 파일 1개)
- "완료" 보고를 그대로 믿지 말고 실제 코드 읽기
